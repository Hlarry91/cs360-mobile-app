Overview

This project is an Android inventory application designed to support basic inventory management tasks. The app allows users to log in, create accounts, view inventory items, and receive SMS notifications related to inventory status. The goal of the app was to create a simple, user-friendly interface while demonstrating core mobile development concepts.

User Needs and Goals

This app was designed to address the need for a simple and organized way to manage inventory. Users need to be able to log in securely, view items clearly, and understand inventory status such as in stock, low stock, or out of stock. The app focuses on clarity, ease of use, and straightforward navigation.

UI Design and Features

The app includes three main screens: a login screen, an inventory screen, and an SMS permission screen. The login screen allows users to enter credentials and create accounts. The inventory screen displays items with their quantities and status, along with options to add or delete items. The SMS screen allows users to grant or deny notification permissions. The UI was designed to be simple, readable, and consistent across screens to improve usability.

Development Approach

The development process followed a step-by-step approach. I started with UI design, then implemented functionality such as login validation, database interactions, and navigation between screens. I used a modular approach by separating responsibilities into different classes like MainActivity, DatabaseActivity, and DatabaseHelper. Testing and debugging were done frequently to resolve issues early, especially with XML and Java integration.

Testing and Functionality

I tested the app using the Android emulator by verifying login behavior, account creation, screen navigation, and button interactions. This process was important because it helped identify issues such as UI mismatches and logic errors. Testing ensured the app functioned as expected and improved overall reliability.

Challenges and Solutions

One challenge during development was connecting the UI elements in XML to the Java code correctly. Mismatched IDs caused errors during testing. I resolved this by carefully reviewing naming conventions and ensuring consistency between layout files and Java classes. Another challenge was managing multiple screens, which I addressed by using intents to navigate between activities.

Successes

I was particularly successful in building a complete multi-screen application that includes login functionality, database interaction, and user permissions. This project demonstrates my ability to connect UI design with backend logic and create a functional mobile application.
